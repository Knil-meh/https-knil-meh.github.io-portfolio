Project Artifacts overview

For the artifacts, one project was chosen primarily. From the class CS320, I chose the contact service project. This assignment originally has you creating logic to add, update, and remove contacts within a Faux database. And the main purpose of the assignment was to run unit tests to ensure that each of these functions are implemented correctly and handle various edge cases, such as addresses being too long or repeat identifying IDs. For the software and engineering category, I took this system and enhanced various limitations, such as increasing the range of edge cases the unit tests can catch, such as null or non-entry inputs. while also cleaning up each of these unit tests so that repeat test functionality is not ran, and instead one test is run across multiple methods instead of a separate one for each method. In the data structures category, I added a hash-based system for temporary storage in memory. This is to improve lookup speeds for specific contract entries. Lastly, direct functionality was added to be able to create, update, remove, and view these contacts. And largely relating to the database category, I created a SQL database that accesses a local database markdown file that houses this information and stores it so that any created or modified contacts will persist after a system reset, so long as the markdown file exists.
